Medical Report Analyzer – RAG System

Problem Statement

Medical reports are often complex and difficult for non-medical individuals to understand. Patients struggle to interpret terminology, values, and implications, leading to confusion and poor decision-making.

Solution

This project uses a Retrieval-Augmented Generation (RAG) approach to analyze medical reports. It retrieves relevant medical context using vector search (FAISS) and combines it with NLP models to generate clear and contextual explanations.

Key Features

  • 🔍 Semantic search using embeddings
  • 📄 Document retrieval with FAISS
  • 🧠 Context-aware response generation
  • ⚡ Efficient processing of large medical reports

Cloud Integration (Simulation)

The system was treated as a SaaS-based application and simulated using CloudAnalyst to evaluate performance on cloud infrastructure. User workloads and backend processing were mapped to cloud components such as user bases, virtual machines, and data centers.

System Mapping

  • Users uploading reports → User Bases
  • Report processing requests → Requests
  • RAG backend system → Data Centers
  • Processing units → Virtual Machines (VMs)

Simulation & Optimization

Simulations were conducted using different load balancing policies. Initially, the Round Robin policy was used, followed by the Throttled policy to optimize system performance and cost.

Key Findings

  • Throttled policy improved response time
  • Better resource allocation across VMs
  • Reduced operational cost under load

Concept: Cost-Performance Balance

The project focuses on achieving an optimal balance between system performance (low response time) and operational cost through efficient resource allocation in cloud environments.

Tech Stack

Python FAISS Transformers RAG NLP CloudAnalyst

Key Implementation

Designed a RAG-based pipeline for document understanding and simulated its behavior as a cloud-based SaaS application using CloudAnalyst. Evaluated system performance under different load balancing strategies to identify optimal configurations.

Challenges Faced

Ensuring accurate retrieval of relevant context from large datasets was challenging. Additionally, mapping the RAG system into a cloud simulation model and analyzing performance metrics required careful abstraction and system understanding.

Future Improvements

  • Real cloud deployment of RAG system
  • Integration with real medical datasets
  • Improved accuracy using domain-specific models
  • Explainable AI for better trust and transparency
